  • Présentation

    Un serveur Web écrit en CPP98 :

    - protocole HTTP/1.1
    - implementé sur la base des RFC 7230-7235
    - connexion server/client par socket TCP/IP
    - utilise select() pour gérer dans un seul thread/processus les multiples connexions entrantes

    Fichier de configuration :

    - implémenation basée sur la configuration de nginx
    - possibilité de lancer plusieurs virtual servers avec port et hostname différents dans le même processus
    - gestion de pages d'erreur custom, de l'index par defaut, de l'autoindex, de la taille max du body du client

    Un serveur résilent :

    - gestion de tous les cas d'erreurs
    - pas d'interruption du service si erreur du coté client ou server
    - testé avec siege et le testeur de l'intra 42

  • Principales méthodes implementées

    GET "target ressource"

    Utilisée pour l'affichage des pages web statiques (hors CGI) ou dynamiques (CGI)

    HEAD "target ressource"

    Renvoie seulement les headers que l'on aurait obtenu avec une requête GET

    PUT "target ressource" + "body"

    Création d'un fichier contenant le body envoyé par la requete

    POST "target ressource" + "body"

    Utilisée pour passer des données a un script CGI

    DELETE "target ressource"

    Supprimer une ressource sur le server

    OPTIONS "path/asterisk form"

    Permet principalement de renseigner sur les methodes autorisees pour une location donnée

    TRACE "path/asterisk form"

    Permet de s'assurer de la bonne réception d'une requête en la renvoyant inchangée

  • Les scripts CGI

    Un script CGI est un programme exécuté par le serveur web, qui traite des données envoyées par le client (formulaire html, par exemple), et génère des réponses (page html, par exemple). Cela permet de proposer au client un service interactif.

    Le serveur est responsable de gérer la connexion, le transfert de données, le transport et les problèmes réseau relatifs à la requête du client. Le script CGI s’occupe quant à lui des problèmes niveau application : accès aux données et traitement des documents.

    Testé avec les scripts habituels : php, perl, bash

    Métavariables traitées

    AUTH_TYPE - CONTENT_LENGTH - CONTENT_TYPE - GATEWAY_INTERFACE - PATH_INFO - PATH_TRANSLATED - QUERY_STRING - REMOTE_ADDR - REMOTE_IDENT - REMOTE_USER - REQUEST_METHOD - REQUEST_URI - SCRIPT_NAME - SERVER_NAME - SERVER_PORT - SERVER_PROTOCOL - SERVER_SOFTWARE
